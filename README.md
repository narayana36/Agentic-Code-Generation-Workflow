# Guided Component Architect

## Overview
This project implements an **Agentic Code Generation Workflow** that transforms natural language descriptions into valid, styled Angular components. It demonstrates how to combine **LLM code generation**, **design system enforcement**, and **self-correction loops** — the building blocks of platforms like Lovable or Bolt.new.

---

## Architecture Overview
```bash
User Prompt
    ↓
Generator (Gemini AI + Design Tokens)
    ↓
Validator (Python Linter-Agent)
    ↓
Agentic Loop (Retries until valid)
    ↓
Output (Angular Component in /output)
```

## How to Run
### 1. clone or Download
``` bash
https://github.com/narayana36/Agentic-Code-Generation-Workflow
```

### 2. Install Dependencies
```bash
pip install -r requirements.txt
``` 

### 3. Configure Environment
Create a **.env** file for Gemini ai api key :
```bash
GEMINI_API_KEY=your_api_key_here
```

### 4. Ensure Design System Exists
Check **design/design-system.json** :
```bash
{
  "primary-color": "#6366f1",
  "secondary-color": "#f59e0b",
  "border-radius": "8px",
  "font-family": "Inter, sans-serif",
  "shadow": "0 4px 30px rgba(0,0,0,0.1)"
}
```

### 5. Run Agentic Loop
```bash
python src/agentic_loop.py
```

### 6. Expected Output
- Angular component code generated by Gemini.
- Validator checks compliance with design tokens.
- Agentic loop retries until valid.
- Final component saved in output/login-card.component.ts

### Input:
user prompt:
```bash
A login card with a glassmorphism effect
```
